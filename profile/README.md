### The idea

Most AI tools optimize for a fluent answer. Perch optimizes for an answer you can verify. The checks run as computation, not as a model guess, and every exception comes back with the records behind it so a person can confirm it. A confident wrong answer is worse than an honest "I could not verify this," so we build for the second one.

### What it does

- **Reads a folder and finds what matters.** Duplicate payments, mismatched invoices, vendors that were never approved, totals that do not reconcile, each flagged with the evidence.
- **Drafts with sources.** Memos, briefs, and summaries where every claim traces back to the material, with unsupported claims marked instead of slipped in.
- **Verifies citations.** Confirms that cited authority exists, says what it is cited for, and is still good.
- **Runs tasks end to end.** Describe a task in the terminal and Perch plans and executes it, from one-off scripts to whole builds.

### Two working styles, one workspace

- **Saffron** reads files under pressure: audits, reconciliations, and analysis where the goal is to find the thing that matters.
- **Quill** writes with sources: memos, briefs, and drafts that have to survive contact with the record.

### Three surfaces, one account

Web for the fastest start, a desktop operator that works from your local folders, and a CLI that brings the same engine to your terminal. Sign in once. Memory and usage carry across all three.

```bash
npm install -g perchai-cli
perch run "research this folder and tell me what matters"
```
